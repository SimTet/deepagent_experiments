# AI Use Case Ideation Canvas

This canvas guides the capture of structured information about an AI idea for evaluation and planning.

---

## Section 1: Stakeholders

### Q1.1 - Problem Owner
**Question:** Who is the problem owner? (Name and role of the person who owns the business problem this AI will solve)
**Type:** text
**Required:** true

### Q1.2 - Budget Owner
**Question:** Who is the budget owner? (Name and role of the person responsible for funding this initiative)
**Type:** text
**Required:** true

### Q1.3 - Technical Owner
**Question:** Who is the technical owner? (Name and role of the person responsible for technical implementation)
**Type:** text
**Required:** true

### Q1.4 - Executive Sponsor
**Question:** Who is the executive sponsor? (Name and role of the senior leader sponsoring this initiative)
**Type:** text
**Required:** true

---

## Section 2: Problem & Goal

### Q2.1 - Problem Statement
**Question:** What problem are you trying to solve? Describe the current pain point or challenge in detail.
**Type:** text
**Required:** true

### Q2.2 - Goal Statement
**Question:** What is the desired outcome? Describe what success looks like when this AI solution is implemented.
**Type:** text
**Required:** true

### Q2.3 - Success Metrics
**Question:** What are 2-3 measurable success metrics? (e.g., reduce processing time by 30%, increase accuracy to 95%)
**Type:** text
**Required:** true

### Q2.4 - Scope Boundaries
**Question:** What is explicitly out of scope for this initiative? (Optional - helps set clear boundaries)
**Type:** text
**Required:** false

---

## Section 3: Value Proposition

### Q3.1 - Value Type
**Question:** What is the primary type of value this AI will deliver?
**Options:** Cost Reduction, Revenue Increase, Risk Mitigation, Efficiency, Compliance, Customer Experience, Other
**Type:** choice
**Required:** true

### Q3.2 - Qualitative Benefits
**Question:** What are the qualitative benefits? (e.g., improved employee satisfaction, better customer insights, faster decision-making)
**Type:** text
**Required:** true

### Q3.3 - Quantitative Benefits
**Question:** What are the quantitative benefits with estimates? (e.g., save $500K/year, reduce manual work by 1000 hours/month)
**Type:** text
**Required:** true

### Q3.4 - Time to Value
**Question:** What is the expected time to realize value after deployment?
**Options:** 0-3 months, 3-6 months, 6-12 months, 12+ months
**Type:** choice
**Required:** true

## Section 5: Data Requirements

### Q5.1 - Data Sources
**Question:** What are the data sources? List internal systems and external sources that will provide data.
**Type:** text
**Required:** true

### Q5.2 - Data Types
**Question:** What types of data will be used?
**Options:** Structured, Unstructured, Mixed
**Type:** choice
**Required:** true

### Q5.3 - Data Content Description
**Question:** Describe the data content. What information do the datasets contain? (e.g., customer transactions, support tickets, sensor readings)

**Required:** true

### Q5.4 - Data Volume Estimate

**Type:** text
**Required:** true

### Q5.5 - Data Availability

**Type:** text
**Required:** true
